센서 수신 모델 ERD — 방식 A에서 B로

device 쪽 연결만 표시했다(사용자·권한 테이블 생략). 핵심 전환은 「채널 = Device」를 「물리 Device ─ 채널 ─ 관측 batch」로 풀어내 같은 시점에 관측된 값 묶음을 스키마가 직접 보존하는 것이다.

현행 AS-IS · 방식 A

측정 채널 하나가 Device 한 건. 요청마다 scalar 하나가 sensor_data 한 행이 되고, 동시 관측 관계는 어디에도 남지 않는다.

변경 TO-BE · 방식 B

Device는 물리 노드(CMAPSS-U1, CNC-EXP01)가 되고, 측정 정의는 sensor_channel로, 관측 한 번은 measurement_batch로 내려간다. 원본 한 행(cycle·100ms 샘플)이 batch 하나에 대응한다.

    zones ||--o{ device : ""
    device ||--o{ sensor_channel : "채널 = 행 등록, DDL 불필요"
    device ||--o{ measurement_batch : "관측 1회 = batch 1행"
    measurement_batch ||--o{ sensor_reading : ""

    device {
        bigint id PK
        varchar code UK  "CMAPSS-U1, CNC-EXP01"
        varchar name
        varchar location
        int expected_interval_seconds
    }
    sensor_channel {
        bigint id PK
        bigint device_id FK
        varchar code  "s4, X1_ActualPosition"

        varchar quantity_kind  "soft 태그 - enum 아님"
        double threshold_value
        varchar threshold_direction
    }
    measurement_batch {
        bigint id PK
        bigint device_id FK

        bigint source_seq  "cycle 번호, 행 인덱스"
    }
    sensor_reading {
        bigint id PK
        bigint batch_id FK
        bigint channel_id FK
        double value  "UNIQUE(batch_id, channel_id)"
    }
    device_status {
        bigint device_id PK
        timestamptz last_seen_at  "노드 단위 freshness"
    }
    channel_status {
        bigint channel_id PK
        boolean in_alarm
        timestamptz last_alert_at
    }

        bigint id PK

신설: sensor_channel · measurement_batch · sensor_reading · channel_status 이동·제거: device.type, device.threshold_value, in_alarm/last_alert_at

필드가 어디로 가나

현행 (A)변경 (B)비고
device = 채널 1개 device = 물리 노드, NEW sensor_channel = 측정 정의 데모 장치 7개 → 물리 device 3개 + 채널 7개
device.type enum sensor_channel.unit + quantity_kind 노드에 온도·압력·진동이 섞이므로 "노드의 타입"은 성립 안 함. enum 강제도 제거
device.threshold_value 이동 sensor_channel.threshold_value (+ direction) 임계값은 채널 속성. 알람 판정 경계도 채널로
sensor_data 1행 = scalar 1개 NEW measurement_batch 1행 + sensor_reading N행 같은 batch의 값은 observed_at·source_seq를 공유 — 동시 관측 복원 가능
sensor_data.recorded_at = 수신 시각뿐 observed_at(원본) / received_at(서버) 분리 A 실측에서 같은 cycle 짝이 1µs 어긋나던 문제 해소
device_status.in_alarm, last_alert_at 이동 NEW channel_status device_status는 노드 freshness(last_seen_at)만 유지
alert.device_id alert.device_id(유지, nullable) + NEW channel_id + batch_id 임계 alert는 device·channel·batch 모두 세팅. freshness(수신 끊김) alert는 채널·batch가 없어 device_id만 세팅
failed_reading (요청 단위) 유지 + 채널 단위 부분 실패 기록 모르는 채널 코드는 부분 실패로 남기고 나머지는 저장

수신 payload 비교

A — 채널마다 독립 요청 ×N

POST /sensor-data
{
  "deviceId": 1,
  "value": 1400.6
}
POST /sensor-data
{
  "deviceId": 2,
  "value": 47.47
}

B — 관측 1회 = 요청 1건

POST /sensor-data
{
  "deviceCode": "CMAPSS-U1",
  "observedAt": "2026-07-17T15:00:00Z",
  "sourceSeq": 12,
  "measurements": {
    "s4": 1400.6,
    "s11": 47.47
  }
}
보류한 확장점 — reading의 비숫자 값(value_text), batch의 원본 보존(context JSONB): CNC의 Machining_Process(문자열) 같은 사례가 실제로 필요해질 때만 추가. 사용자·권한 테이블(users, zone_users, refresh_tokens)은 이 변경의 영향 밖이라 생략.
